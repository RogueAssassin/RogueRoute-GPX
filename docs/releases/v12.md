# RogueRoute GPX v12

v12 is the standalone Docker, routing reliability, and compact-GPX release.

## Highlights

- Publishes the web application to
  `ghcr.io/rogueassassin/rogueroute-gpx:v12` for AMD64 and ARM64.
- Adds a standalone Compose deployment with its own network and no dependency
  on Rogue Dashboard, `media-net`, Node.js, or pnpm on the server.
- Raises the bounded strict-routing snap maximum from 1,500 m to 5,000 m after
  real-world testing resolved distant Auckland portal routing failures.
- Adds adaptive track simplification, targeting 1,000 points by default while
  retaining waypoints, leg boundaries, endpoints, and meaningful bends.
- Adds an interactive OpenStreetMap route preview with route, waypoint,
  auto-snap, and failure overlays.
- Fixes false OSM download-validation failures caused by `.osm.pbf.part`
  filenames and recovers valid preserved downloads without transferring them
  again.
- Fixes modern OSRM MLD graph readiness checks and makes multi-region
  preparation continue, retry safely, and report per-region results.
- Updates the supported toolchain to Node.js 24.18.0, pnpm 11.12.0,
  Next.js 16.2.10, React 19.2.7, and TypeScript 6.0.3.

## Deployment order

1. Upload and commit the v12 source package to GitHub.
2. Create the `v12` release tag. The GHCR workflow builds and publishes the
   container automatically.
3. Confirm the package is public, or log the server in with a read-packages
   token.
4. Run the standalone installer on the server.

The installer moves the previous application directory to a timestamped
backup. It never deletes external OSRM data such as `/mnt/h/osrm`.
